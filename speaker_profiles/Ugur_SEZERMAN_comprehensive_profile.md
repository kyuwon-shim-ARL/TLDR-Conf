### [1장: 도시는 살아있다 - 지하철 미생물 지도 프로젝트]

#### 이야기의 시작
2021년, Cell 저널에 실린 한 연구가 세계를 놀라게 했습니다. 60개 도시, 3년간, 지하철과 대중교통 시스템에서 수집한 4,728개 샘플을 분석한 결과...

**발견**:
- 10,928종의 바이러스
- 1,302종의 박테리아
- 838,532개의 CRISPR 배열 (기존 데이터베이스에 없던 것들!)
- 4,246종의 알려진 미생물 + 수많은 미지의 종

#### 왜 지하철인가?
지하철은 **도시의 혈관**입니다. 수백만 명이 매일 오가며, 전 세계에서 온 사람들이 섞입니다. 지하철 손잡이, 의자, 벽면에는 **도시의 미생물 생태계**가 압축되어 있습니다.

**비유**:
```
지하철 = 도시의 단면도
손잡이 표면 = 미생물 타임캡슐
swab 샘플 = 도시 건강 체크업
```

#### 핵심 발견 1: 도시마다 고유한 미생물 지문
- 뉴욕, 서울, 파리, 도쿄... 각 도시는 **독특한 미생물 시그니처**를 가짐
- 기후, 지리, 인구 구성이 결정 요인
- 31종의 "코어 미생물" = 97% 도시에서 공통 발견
- 이들은 **인간 공생 미생물과 다름** (환경 적응종)

#### 핵심 발견 2: 항생제 내성 유전자의 세계 지도
- 도시마다 AMR 유전자 **프로필이 다름**
- Type과 Density가 극명한 차이
- 어떤 도시는 "AMR 핫스팟"
- 공중보건 개입의 **우선순위 설정** 가능

**실전 적용**:
```
Step 1: 샘플 수집 (지하철 손잡이 swab)
Step 2: DNA 추출 & 메타게노믹 시퀀싱
Step 3: 바이오인포 파이프라인 분석
Step 4: AMR 유전자 프로파일링
Step 5: 도시 미생물 지문 생성
```

**결과**:
- 감염병 발생 예측
- AMR 확산 모니터링
- 공중보건 정책 수립

### [2장: 병원은 전쟁터다 - 항생제 내성 유전자의 지도]

#### 배경
병원은 AMR의 **진원지**입니다. 항생제 사용이 가장 많고, 면역이 약한 환자들이 밀집되어 있으며, 병원체들이 **생존 압력**을 받는 곳입니다.

#### SEZERMAN의 접근: "Cartography" (지도 제작)
일반적 연구: 특정 균주 분석
SEZERMAN: 병원 **전체 환경**을 3D 지도로

**연구 디자인** (Nature Medicine 2020):
```
Target: 3차 병원 (Tertiary Hospital)
Samples:
├─ 병실 표면 (침대, 의자, 문손잡이)
├─ ICU 장비 (인공호흡기, 모니터)
├─ 수술실 환경
├─ 병원 공기 샘플
└─ 환자 샘플 (침습적/비침습적)

Analysis:
├─ Metagenomics (전체 미생물 군집)
├─ AMR gene profiling (내성 유전자)
├─ Opportunistic pathogen tracking
└─ Spatial mapping (공간 분포)
```

#### 핵심 발견
**발견 1**: AMR 유전자 "핫스팟"
- ICU > 일반 병실 > 외래
- 고위험 지역 식별 가능
- **근거 기반 방역** 가능

**발견 2**: Opportunistic Pathogens의 숨은 서식지
- Pseudomonas aeruginosa: 수도꼭지, 샤워헤드
- Acinetobacter baumannii: 침대 난간, 키보드
- Clostridioides difficile: 화장실, 바닥

**발견 3**: 유전자 수평 이동 (HGT) 증거
- 병원 내 다른 균종 간 AMR 유전자 공유
- Plasmid, Transposon 매개
- **내성 확산의 실시간 증거**

#### 비유: 병원 = 미생물 전쟁터
```
항생제 = 폭격
생존 균 = 내성 획득 병사
병원 환경 = 전쟁터 지형
AMR 유전자 = 무기 거래
```

**적용 가능성** (한국 파스퇴르):
1. 한국 병원 AMR 지도 제작
2. 고위험 병원체 추적
3. 방역 프로토콜 최적화
4. Early warning system 구축

### [3장: 길을 찾는 알고리즘 - pathfindR의 혁신]

#### 문제 정의
유전체 연구의 고질적 문제:
```
Input: 수천 개의 유전자 리스트
        (예: 암 vs 정상, 처리 vs 대조)

Question: 이 유전자들이 의미하는 생물학적 현상은?

Challenge: 유전자 하나하나는 의미 없음
           → **Pathway (경로)** 수준에서 이해해야
```

#### 기존 방법의 한계
**Traditional Enrichment Analysis**:
1. 유전자 리스트를 받음
2. 각 pathway에 몇 개나 포함되는지 계산
3. 통계적 유의성 검정
4. 유의한 pathway 리스트 출력

**문제**:
- 유전자 간 **상호작용 무시**
- Protein-protein interaction (PPI) 정보 미활용
- 약한 신호 (weak signal) 놓침
- False positive 많음

#### pathfindR의 혁신: Active Subnetwork
**핵심 아이디어**:
유전자는 혼자 작동하지 않는다!
→ **Protein-protein interaction network**에서 **활성화된 부분 네트워크** (active subnetwork)를 찾자

**비유**:
```
기존 방법 = 도시에서 "범죄 발생 주소" 리스트만 보기
pathfindR = 범죄 발생 지역을 **지도에 표시**하고,
            인접한 지역들도 함께 분석
            → 범죄 "핫스팟" (active subnetwork) 발견
```

#### 알고리즘 단계
```
Step 1: 입력 유전자 리스트 (differential expression)
         ↓
Step 2: PPI 네트워크에 매핑
         ↓
Step 3: Active subnetwork 탐색
        - Greedy algorithm
        - Simulated annealing
        - Genetic algorithm
         ↓
Step 4: 각 subnetwork에서 pathway enrichment
         ↓
Step 5: 여러 subnetwork 결과 통합
         ↓
Step 6: Pathway clustering (유사한 pathway 묶기)
         ↓
Step 7: Pathway activity 점수 (각 샘플별)
```

#### 실제 예시: 암 연구
**Case**: 폐암 vs 정상 조직 RNA-seq
```
Input: 2,547개 차별 발현 유전자

기존 방법:
- "Cell cycle" pathway 발견 (p < 0.001)
- "끝"

pathfindR:
- Active subnetwork #1: TP53-MDM2-CDKN1A axis
  → "p53 pathway in cancer" (더 구체적!)
- Active subnetwork #2: EGFR-MAPK-ERK cascade
  → "EGFR tyrosine kinase inhibitor resistance"
  → **임상 치료 타겟 제시!**
- Active subnetwork #3: PD-L1-CD274 cluster
  → "PD-1 checkpoint pathway"
  → **면역치료 적용 가능성!**
```

**결과**:
- 문헌에서 지지되는 pathway 우선 순위 상승
- 다른 도구가 놓친 pathway 발견
- **질병 이름과 일치하는 pathway** 식별 (높은 정확도)

#### 오픈소스 임팩트
```

├─ Urban microbiome (monthly)
├─ Clinical isolates (weekly)
├─ Wastewater (daily)
└─ Travel/immigration data

    ↓
AI/ML Pipeline:
├─ Anomaly detection (AMR spike)
├─ Outbreak prediction (LSTM)
├─ Source tracking (network analysis)
└─ Spread simulation (agent-based model)

    ↓
Action Platform:
├─ Alert to hospitals
├─ Public health dashboard
├─ Policy recommendation
└─ Intervention evaluation
```
